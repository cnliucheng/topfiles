# TopFiles 安全与可靠性修复设计

## 目标

消除公开分享的同源脚本执行风险，恢复可靠的服务端测试，防止自动保存覆盖较新的编辑内容，并降低首次进入编辑器的下载量。

## 已确认的产品决策

- 所有 `/u/:filename` 分享内容均作为 `text/plain; charset=utf-8` 返回，不提供浏览器内联预览。
- 保留 CodeMirror 6；不改用 Monaco 或 Shiki。编辑器组件在首次使用时异步加载，并按文件语言加载其扩展。

## 服务端安全

分享路由不再根据文件 MIME 类型输出可执行或可渲染内容。响应固定为纯文本，并保留 `X-Content-Type-Options: nosniff`、安全的文件名和短时公共缓存。

生产环境要求同时存在 `JWT_SECRET` 与 `COOKIE_SECURE=true`，否则服务拒绝启动。这样部署错误不会降级为不安全的会话 Cookie。

## 测试资源生命周期

测试上下文显式关闭 HTTP server 和 SQLite 数据库后，才删除 WAL、SHM 与数据库文件。单个 server 在一个测试中只启动一次；请求助手等待关闭完成，避免端口和 SQLite 文件锁竞态。

新增公开 JavaScript 分享内容必须返回纯文本的回归测试，并让原有服务端全套测试可稳定完成。

## 自动保存一致性

本地草稿仍即时写入 localStorage。云端自动保存改为带短暂防抖的单一串行队列：编辑期间合并为最新文本；前一次保存完成后才提交最新待保存快照。显式“保存到云端”继续直接提交，并使队列不再写入过期内容。

## 前端加载

编辑器外层使用 Vue 异步组件。未进入编辑区域时不下载 CodeMirror；加载后保持现有的快捷键、右键菜单和语言切换行为。语言扩展继续按当前文件类型动态导入。

## 验收

- 分享 `.js` 文件的 `Content-Type` 为 `text/plain; charset=utf-8`。
- `npm test` 在服务端完整通过，不再出现超时或遗留 WAL 文件。
- 快速连续编辑后，最后一次内容是最终云端版本。
- 缺少安全生产配置时服务启动失败。
- 前端构建通过，初始入口不再包含 CodeMirror 编辑器代码。
