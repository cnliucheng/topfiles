# TopFiles 苹果风工作区 UI 设计

## 目标

将现有界面重构为克制、内容优先的苹果原生感工作区，同时保留全部编辑、登录、云端文件、分享、导入与移动抽屉能力。

## 视觉系统

- 页面基色使用暖白与石墨灰，减少纯白大块和高对比边界。
- 蓝色仅用于主操作、选中态、键盘焦点与必要的状态提示；不再作为大面积品牌底色。
- 使用 12–20px 圆角、细半透明边框与两级轻阴影表达层级；暗黑模式采用深石墨表面而非反色。
- 正文维持系统字体；编辑器继续使用等宽字体。普通交互元素的触控高度不低于 44px。

## 工作区

桌面端为两栏布局。左栏按账户、文件标题与数量、主“新建文件”按钮、圆角文件行、设置/退出分组排列。编辑区成为独立工作台，标题、保存状态、分享和下载使用轻量胶囊按钮，代码内容占据主视觉。

窄屏使用磨砂遮罩下的抽屉式文件栏，底部留出安全区域；打开、关闭、删除与文件选择均保持 44px 以上可触达区域。

## 弹窗与状态

登录、注册、导入、分享、账号设置与确认弹窗统一为浮层卡片：暖白表面、石墨文本、细描边、低阴影、明确的键盘焦点。错误使用语义红色，成功和已保存状态使用文字/图标而非只依赖颜色。

## 验收

- 桌面端视觉不再含大面积品牌蓝，蓝色仅用于主操作与选中/焦点态。
- 侧栏和编辑器具备明确但低干扰的层级与留白。
- 375px 窄屏没有横向滚动，抽屉与主要按钮可触达。
- 现有登录、文件、保存、分享和导入行为保持不变。
- `npm run build` 通过。
